# Projektfortschritt & Assessment

Diese Datei hilft dir, deinen Lernfortschritt zu verfolgen und zu reflektieren.

## Self-Assessment Checkliste

### Programmierkonzepte

**Grundlagen**
- [ ] Ich verstehe, was Variablen sind
- [ ] Ich kann verschiedene Datentypen verwenden (String, Number, Boolean)
- [ ] Ich verstehe die Konsole/Terminal-Ausgaben

**Kontrollstrukturen**
- [ ] Ich kann If-Else Bedingungen schreiben
- [ ] Ich verstehe Vergleichsoperatoren (==, !=, <, >)
- [ ] Ich kann logische Operatoren verwenden (AND, OR, NOT)
- [ ] Ich kann verschachtelte Bedingungen erstellen

**Schleifen**
- [ ] Ich kann For-Schleifen schreiben
- [ ] Ich kann While-Schleifen schreiben
- [ ] Ich verstehe, wann ich welche Schleife verwende
- [ ] Ich kann Schleifen mit break/continue steuern

**Funktionen**
- [ ] Ich kann eigene Funktionen definieren
- [ ] Ich verstehe Parameter und Argumente
- [ ] Ich kann Return-Werte verwenden
- [ ] Ich kann Funktionen dokumentieren

**Datenstrukturen**
- [ ] Ich kann Arrays/Listen erstellen und verwenden
- [ ] Ich kann Objekte/Dictionaries verwenden
- [ ] Ich kann durch Collections iterieren
- [ ] Ich verstehe Verschachtelungen

### Technische Skills

**Entwicklungsumgebung**
- [ ] Ich kann VS Code effektiv nutzen
- [ ] Ich verstehe die Ordnerstruktur meines Projekts
- [ ] Ich kann Dateien erstellen und organisieren

**Debugging**
- [ ] Ich kann Fehlermeldungen lesen und verstehen
- [ ] Ich kann einfache Fehler selbst beheben
- [ ] Ich nutze console.log/print zur Fehlersuche
- [ ] Ich frage gezielt um Hilfe bei komplexen Problemen

**Version Control (optional)**
- [ ] Ich verstehe Git Grundbegriffe
- [ ] Ich kann Commits erstellen
- [ ] Ich kann mit GitHub arbeiten

### Problemlösungs-Fähigkeiten

**Planung**
- [ ] Ich kann ein Problem in kleinere Schritte zerlegen
- [ ] Ich erstelle Pseudo-Code vor der Implementierung
- [ ] Ich plane die Struktur meines Codes

**Recherche**
- [ ] Ich weiß, wie ich nach Lösungen suche
- [ ] Ich kann Dokumentation lesen und verstehen
- [ ] Ich stelle gute Fragen an Copilot/Lehrer

**Code-Qualität**
- [ ] Ich schreibe lesbaren Code
- [ ] Ich nutze aussagekräftige Variablennamen
- [ ] Ich kommentiere meinen Code sinnvoll
- [ ] Ich teste meinen Code

## Finale Projekt-Bewertung

### Technische Bewertung

**Funktionalität** (Erfüllt das Projekt die Anforderungen?)
- [ ] Alle Must-Have Features funktionieren
- [ ] Code läuft ohne Fehler
- [ ] Benutzerfreundlich

**Code-Qualität** (Ist der Code gut geschrieben?)
- [ ] Gut strukturiert und organisiert
- [ ] Lesbar und verständlich
- [ ] Sinnvoll kommentiert
- [ ] Wiederverwendbare Funktionen

**Fehlerbehandlung** (Reagiert das Programm gut auf Fehler?)
- [ ] Ungültige Eingaben werden abgefangen
- [ ] Hilfreiche Fehlermeldungen
- [ ] Programm stürzt nicht ab

### Lern-Bewertung

**Konzepte verstanden:**
- [ ] Alle Lernziele erreicht
- [ ] Kann Konzepte erklären
- [ ] Kann Konzepte auf neue Probleme anwenden

**Selbstständigkeit:**
- [ ] Probleme eigenständig gelöst
- [ ] Effektiv nach Hilfe gesucht
- [ ] Eigene Lösungsansätze entwickelt

**Reflexion:**
- [ ] Versteht eigene Stärken
- [ ] Kennt Bereiche für Verbesserung
- [ ] Hat konkrete nächste Lernziele
